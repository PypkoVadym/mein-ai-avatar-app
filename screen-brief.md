## ⚠️ Ключевые изменения после CEO-разговора (31 марта)

Эти пункты **полностью меняют приоритеты** по сравнению с предыдущей версией брифа:

1. **Create Character = первое и главное действие**, не второстепенный шаг
2. **Биометрика ЕСТЬ в MVP** — пользователь загружает 2–3 фото + видео поворота головы (ранее было "не в MVP")
3. **Presets = seeds**, не готовые аватары. Пользователь выбирает seed → генерируется уникальный персонаж похожий на референс
4. **Время генерации** — проектируем под конкурентные значения рынка (HeyGen и аналоги). Текущий технический лимит — временный, endpoint будет заменён без изменения UX.
5. **Freeform-поле** для ввода контент-идеи + optional поля (ссылки, хэштеги) — для логирования, не обработки
6. **Без онбординга** — простой продукт, прямая mobile закупка, данные через freeform
7. **Compliance = конкурентное преимущество**, а не барьер. Подавать как заботу о пользователе

## Продукт

**TWIN** — мобильное приложение для генерации AI-видео с персональным AI-аватаром.

Ключевое ценностное предложение: **пользователь создаёт себя один раз → потом бесконечно генерирует видео только из текста.**

- Платформа: **iOS + Android** (cross-platform)
- Режим: **light mode**
- Навигация: **bottom tab bar** — Generate | Inspiration | Library
- UX-принцип: **если флоу создания персонажа работает = продукт работает. Если нет = провал.**
- Рыночный gap: HeyGen и аналоги всё ещё сложны для массового пользователя (гуманитарий, без tech-опыта). TWIN → c-маркет.

## Целевой пользователь

**Контент-мейкеры** — ведут TikTok, Instagram Reels, YouTube Shorts.
Делают контент для личного бренда или продвижения продукта/услуги.

**Пример:** Dimitris — ведёт instool с плохим английским, хочет полировать контент с AI, но существующие инструменты дают плохой результат — время тратится, качество низкое.

**Боль:** нет понятного флоу создания персонажа → нет retention → нет продукта.

---

## Визуальный стиль

- Референсы: **Instagram, TikTok, Pinterest**
- Чистый, эстетичный, современный — аудитория привыкла к качественному контентному UI
- Собственной дизайн-системы нет → стиль через референсы

## Полный MVP-флоу

```
Signup (Age Gate + ToS)
        ↓
CREATE CHARACTER ← ключевой экран, первое действие
  ├── Path A: "Создай себя" (2–3 фото + видео поворота головы + consent)
  └── Path B: Выбери seed (preset-образ как референс → генерация похожего)
        ↓
   Генерация аватара (~до 40 мин → Library pending)
        ↓
GENERATE (скрипт → видео)
  ├── Freeform промпт + optional поля (ссылки, хэштеги) — логирование
  ├── Формат контента (TikTok / Instagram / TED Talk / лекция / infographic)
  ├── Количество вариаций (1 / 2–3 для A/B)
  └── Confirmation + Moderation check
        ↓
   Генерация видео (~до 40 мин → Library pending + push)
        ↓
Library → Result + Download (с compliance чекбоксами)

Cookie Banner (первый запуск)
Coming Soon — demand signal (URL, hashtag, file — только логирование)
```

## Экраны

### 0. Cookie Consent Banner
**Назначение:** соответствие ePrivacy Directive (ЕС). Появляется при первом запуске.

**Элементы:**
- Текст: «Мы используем только необходимые cookies. Без трекинга.»
- Кнопка: **Принять**
- Ссылка на Privacy Policy

**Compliance:**
- Только essential cookies. Никаких аналитических SDK на клиенте. Server-side logging.

---

### 1. Регистрация (Signup + Age Gate + ToS Clickwrap)
**Назначение:** аккаунт, блокировка несовершеннолетних, фиксация согласия.

**Элементы:**
- Email, Пароль
- Дата рождения + подпись: «Только для проверки возраста (18+). Не сохраняется.»
- Чекбокс ToS + Privacy Policy (активные ссылки)
- Кнопка **Создать аккаунт** (disabled до чекбокса)

**Состояния:**
- < 18 лет → дружелюбный блок, никаких данных не сохраняется, нет обходного пути

**Compliance:**
- Clickwrap: логировать timestamp, IP, версию ToS
- Хранить только `is_age_verified: true` + timestamp. Дату рождения не хранить.

---

### 2. CREATE CHARACTER ← ключевой экран
**Назначение:** первое и главное действие пользователя. Создание персонального AI-аватара — либо себя, либо на основе seed-образа.

**Два пути:**

#### Path A: «Создай себя»
Пользователь загружает минимальный биометрический input:
- **2–3 фотографии** (разные углы, освещение)
- **Короткое видео** поворота головы (15–30 сек)

**UX-фрейминг (от Rustam):** подать запись видео как «последний раз, когда тебе нужно это делать». Воодушевляющий текст. Compliance — как забота, не как бюрократия.

**Consent flow** (обязателен для биометрики):
- Явный экран с объяснением зачем нужны данные
- Чекбокс: «Я даю согласие на обработку моих биометрических данных для создания AI-аватара»
- Чекбокс: «Я понимаю, что созданный аватар буду использовать только я»
- Кнопка **Продолжить** только после обоих чекбоксов

#### Path B: «Выбери seed-образ»
Пользователь выбирает preset — не готовый аватар, а **референс-изображение** для генерации.
Генератор создаёт персонажа, **похожего на seed, но уникального** (continuity через seed).

**По аналогии с концепцией Rustam:** набирается набор кусочков промпта вокруг seed — итог похож на референс, но кастомизированный.

**Элементы экрана:**
- Заголовок: «Создай своего персонажа»
- Две карточки выбора пути (крупные, визуальные): «Это я» / «Выбрать образ»
- Path A: upload-зона + инструкция по фото/видео
- Path B: сетка seed-карточек (статичные фото, разнообразные) + поле имени/описания персонажа
- После выбора пути → кнопка **Создать персонажа**
- После запуска → redirect в Library, pending-карточка аватара

**Compliance (биометрика в MVP):**
- GDPR Art. 9 + BIPA: явный consent на биометрические данные (фото, видео)
- Видео и фото обрабатываются только для генерации аватара, не хранятся после обработки
- Consent-экран **до** upload — пользователь должен принять условия перед загрузкой данных
- Логировать: timestamp согласия, IP, версию privacy policy
- Документировать: «Seed-пресеты — вымышленные образы без сходства с реальными людьми»
- Output-side likeness check: даже с seed'ами возможны случайные сходства с celebrity → проверка на выходе

### 3. Скрипт + Формат
**Назначение:** ввод контент-идеи, выбор формата, логирование пользовательских запросов.

**Элементы:**
- Header: миниатюра созданного аватара + кнопка «Изменить»
- **Freeform-поле**: «Что хочешь рассказать? Любая идея, тема, тезис» (основной input)
- **Optional поля** (не обязательные, с пометкой «не обязательно»):
  - Ссылка на референс / вдохновение
  - Хэштег / тренд
  → Эти поля логируются как product intelligence. UI не обещает их обработку.
- **Формат контента** (pill-выбор):
  - TikTok (~15–30 сек)
  - Instagram Reel (~20–30 сек)
  - TED Talk / лекция (~2–5 мин)
  - Infographic-стиль
- **Количество вариаций**: 1 видео / 2–3 вариации (для A/B тестирования в рекламе)
- AI-вариант скрипта (опциональная карточка после ввода промпта)
- Кнопка: **Сгенерировать видео →**

**Compliance:**
- Freeform-поле → Content Moderation Pipeline до генерации
- Optional поля: логировать URL/hashtag, не скрапить и не обрабатывать содержимое
- AI-варианты помечены как «Предложение». Disclaimer о точности обязателен.
- FTC safeguard: коммерческий контент → флаг перед генерацией

---

### 4. Confirmation + Модерация
**Назначение:** результат проверки скрипта + управление ожиданием (~40 мин).

**Элементы:**
- Статус проверки (успех / блок)
- Саммари: аватар + формат + количество вариаций
- (условно) FTC-предупреждение для коммерческого контента
- **Важный UX:** явное сообщение «Генерация займёт до 40 минут. Видео появится в Library — уходи, мы пришлём уведомление.»
- Кнопка **Запустить** → немедленный redirect в Library

**UX — async:**
1. Redirect в Library сразу после нажатия кнопки. Никакого экрана ожидания.
2. Push-уведомление при готовности — запросить здесь или в Library.
3. Конкретное время не называть в UI — зависит от backend'а. Показывать прогресс-индикатор.

**Compliance:**
- Логировать модерационное решение: user ID, content hash, timestamp
- FTC-флаг → обязательное предупреждение видимо пользователю

---

### 5. Library (Библиотека)
**Назначение:** список аватаров и видео. Основной экран ожидания.

**Типы карточек:**
- **Pending — аватар:** «Создаётся твой персонаж...» + прогресс-индикатор
- **Готовый аватар:** миниатюра + кнопка «Создать видео»
- **Pending — видео:** «Генерируется...» + прогресс-индикатор
- **Готовое видео:** превью + кнопка «Скачать»

**Push notification prompt:**
Запрашивать сразу после первого запуска генерации (максимально релевантный момент).

**Compliance:**
- Watermark «AI» виден на превью даже в Library
- Видео хранятся до удаления пользователем

---

### 6. Результат + Download
**Назначение:** просмотр готового видео, скачивание с compliance-гейтом.

**Элементы:**
- Превью 9:16 с watermark «AI Generated»
- Если несколько вариаций — свайп между ними
- 2 обязательных чекбокса (+ 3-й FTC при коммерческом контенте)
- Кнопка **Скачать MP4** (disabled до чекбоксов)
- «Пожаловаться на видео»

**Compliance:**
- Watermark: видимый overlay + metadata (ai_generated: true в MP4 XMP/EXIF)
- Логировать download: user ID, video ID, timestamp, IP, флаги чекбоксов
- Имя файла: slug `ai-generated` обязателен
- Report button → abuse@ (EU DSA)

---

### 7. Inspiration
**Назначение:** галерея видео → вдохновение → «Сделать похожее» → начало генерации.

**Элементы:**
- Pill-фильтры: Все | 🔥 Trend | ⚡ Hot | 🌏 Asia | 💄 Beauty | 📈 Business
- Сетка видеокарточек (9:16), badge «AI» на каждой
- CTA «Сделать похожее →» → переход на Create Character / Script с преселектом формата

**Compliance:**
- AI badge на каждом видео обязателен (EU AI Act Art. 50)
- UGC: consent пользователя на публичный показ + те же модерационные проверки

---

### 8. Coming Soon (Demand Signal)
**Назначение:** UI для фич которых нет → логирование интереса как product intelligence.

**Карточки:**
- Ссылка на контент (URL) → логировать URL, не скрапить
- Хэштег / тренд → логировать, не обрабатывать
- Загрузить файл → логировать тип + размер, **файл не принимать**

**Compliance:**
- Раскрыть в Privacy Policy: «Мы логируем попытки взаимодействия с функциями»
- Анонимизировать через 90 дней
- Opt-out по запросу

## Технические ограничения

### Время генерации
- Проектируем под **конкурентные значения рынка** — ориентир HeyGen и аналоги (минуты, не десятки минут).
- Текущий технический лимит — временный, скоро будет неактуален. В UX не закладываем.
- UX поддерживает async-паттерн (Library + push) как универсальный, но не строится вокруг долгого ожидания.
- После замены endpoint'а — никаких изменений в продуктовом UX.

### Compliance checklist (обязательно к шипу)
- [ ] ToS live с clickwrap consent logging
- [ ] Privacy policy с биометрическим разделом (новый, т.к. биометрика теперь в MVP)
- [ ] Age gate < 18
- [ ] Cookie banner
- [ ] Биометрический consent-экран перед upload (GDPR Art. 9 + BIPA)
- [ ] Content moderation на всех freeform inputs
- [ ] IP/trademark detection
- [ ] FTC endorsement safeguard
- [ ] AI-watermark на каждом output (visible + metadata)
- [ ] Download гейтован за 2 чекбокса
- [ ] Download logging
- [ ] Report button → abuse@
- [ ] Response policy задокументирована
- [ ] Seed-пресеты проверены на celebrity resemblance
- [ ] Лицензия TTS в письменном виде
- [ ] Adversarial testing

### Биометрика (теперь в MVP — требует отдельного compliance)
Поскольку создание собственного аватара из фото/видео — **основной флоу MVP**, нужно:
- Явный consent-экран ДО любого upload (не просто чекбокс в ToS)
- Privacy policy с разделом о биометрических данных
- Описать: что собирается, для чего, как обрабатывается, когда удаляется
- Рекомендация: исходные фото/видео удалять после генерации аватара (хранить только аватар)
- GDPR: пользователь должен иметь право удалить аватар и биометрические данные

### Не в MVP
- Voice cloning
- Социальный постинг
- Аналитика / трекинг клиентские
- Маркетплейс персонажей
- Background music
- Upload product images / brand kit
